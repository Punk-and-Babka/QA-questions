# 7. В чем основное преимущество попарного тестирования
Основное преимущество pairwise в том, что он позволяет сильно сократить количество тестов, но при этом сохранить хорошее покрытие комбинаций параметров.

Полный перебор часто невозможен. Например, у нас есть тип клиента, валюта, тип карты, канал операции, способ подтверждения и статус клиента. Если у каждого параметра по несколько значений, полный набор быстро превращается в сотни или тысячи комбинаций.

Pairwise строит набор так, чтобы каждая пара значений встретилась хотя бы один раз. На практике многие дефекты возникают именно на взаимодействии двух параметров, а не всех параметров сразу. Поэтому pairwise даёт хороший баланс между покрытием и затратами.

Например, в банковском приложении можно проверять переводы по комбинациям:

- валюта: RUB / USD / EUR;
- тип клиента: физлицо / ИП / юрлицо;
- канал: web / mobile;
- способ подтверждения: SMS / push;
- тип карты: дебетовая / кредитная.

Полный перебор будет большим, а pairwise поможет сократить набор до разумного количества кейсов.

Но я бы не использовал pairwise слепо. Для критичных сценариев, где есть деньги, безопасность или регуляторные требования, некоторые комбинации нужно добавить вручную по рискам, даже если pairwise их не выбрал.
