# 19. Из каких компонентов состоит тест-кейс
Тест-кейс обычно состоит из нескольких основных частей.

Во-первых, это **название** — короткое и понятное описание того, что проверяется. Например: “Успешный перевод между своими счетами”.

Дальше идут **предусловия**: что должно быть подготовлено до начала проверки. Например, пользователь авторизован, у него есть два активных счёта, на счёте списания достаточно средств.

Потом — **тестовые данные**: пользователь, счёт, сумма, валюта, тип операции, код подтверждения или другие данные, которые нужны для проверки.

Основная часть — **шаги выполнения**. Они должны быть понятными и последовательными.

После каждого важного действия или в конце тест-кейса указывается **ожидаемый результат**. Например: операция получила статус `Success`, баланс изменился корректно, запись появилась в истории.

Также могут быть поля: приоритет проверки, модуль, связанное требование, автор, дата обновления, статус прохождения, комментарии.

Главное — тест-кейс должен быть воспроизводимым. Другой QA должен открыть его, выполнить шаги и получить понятный результат без дополнительных устных объяснений.
